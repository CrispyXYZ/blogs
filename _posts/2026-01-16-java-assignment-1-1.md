## 实体类的实现

本文**不**记录**过于基础**的部分。

开始时有个小插曲，取包名的时候本来写的是 `models`，总感觉不对劲，查了下才知道一般使用的是单数形式，也就是 `model`。

之后编写抽象类 `Person`，突然忘了抽象类能不能有非抽象方法，遂查阅资料，发现可以。抽象类只是可以拥有抽象方法且不能被实例化的类。

此时又有一个疑问，为什么 `Student` 类的成绩 Map 要用课程名 `String` 作为 key，而不是 `Course` 类作为 key？此处让我联想到了 JSON。JSON 通常使用字符串作为 key，这样设计可以方便以后对 Java 对象进行序列化和反序列化。（某个基于 LibGDX 的游戏就是使用的序列化方法用 JSON 存储游戏存档）

本以为这部分会很简单，但是实际上，已经重写了我对 Java 的认知：在 `Person` 类中定义了私有字段 `name`，那么在其子类 `Student` 中用 `name` 访问会直接报错，为什么？既然子类 `Student` 继承了父类 `Person`，那么就应该“复制”父类的字段，然后保持其访问修饰符，这种理解为什么错了？

查阅资料，发现可以这样理解，即对象在继承父类后，确实会拥有父类的字段和方法（的引用），但对象也会创建子类中声明的字段和方法。对某个符号，会优先引用子类的（若父类也有该符号，需用 `super` 关键字引用父类的），其次才是父类的。此时父类的字段是私有的，所以不能被子类访问，就会报错。
