本文将简单介绍 HTTP 中的请求方法、标头和状态码。

## 请求方法

参考：[HTTP 请求方法](https://developer.mozilla.org/zh-CN/docs/Web/HTTP/Reference/Methods)

HTTP 定义了一组请求方法（常被称为 HTTP 动词），用于表示对指定资源要执行的期望操作。每个方法都具有不同的语义，但也共享一些共同特征，例如是否安全、幂等或可缓存。

以下是常用的 HTTP 请求方法及其说明：

|方法|说明|
|--|--|
|`GET`|请求指定资源的表示形式，只应用于获取数据。|
|`HEAD`|请求与 `GET` 相同的响应，但不包含响应体。|
|`POST`|将实体提交到指定资源，常引起服务器状态变化或副作用。|
|`PUT`|用请求中的有效载荷替换目标资源的所有当前表示。|
|`DELETE`|删除指定的资源。|
|`CONNECT`|建立到目标资源标识的服务器的隧道。|
|`OPTIONS`|描述目标资源的通信选项。|
|`TRACE`|沿路径到目标资源执行消息环回测试。|
|`PATCH`|对资源应用部分修改。|

## HTTP 标头

HTTP 标头（Headers）是 HTTP 协议中用于在客户端与服务器之间传递**附加信息**的核心机制。它们以**键值对**的形式存在，构成了请求和响应的元数据部分。通过标头，可以控制缓存行为、进行内容协商、管理连接持久性、实现身份认证、定义内容类型与编码，以及传递各类状态信息等。无论是请求还是响应，其主体内容的具体处理方式在很大程度上都由伴随的标头字段来定义和指导，从而实现了 HTTP 通信的灵活性与丰富功能。

具体详见：[HTTP 标头](https://developer.mozilla.org/zh-CN/docs/Web/HTTP/Reference/Headers)

## HTTP 状态码

HTTP 状态码是服务器对客户端请求作出的标准化的数字响应，它是一个三位数代码，如同服务器发回的简短状态报告。

|类别|范围|类别名称|简要说明|
|----|----|----|----|
|`1xx`|`100-199`|信息响应|表示请求已被接收，需要继续处理。这是一个临时状态，仅告知客户端请求的初始部分已收到，请继续。|
|`2xx`|`200-299`|成功响应|表示请求已成功被服务器接收、理解并处理。这是客户端最希望看到的响应类别。|
|`3xx`|`300-399`|重定向响应|表示需要客户端采取进一步的操作（如访问另一个地址）才能完成请求。|
|`4xx`|`400-499`|客户端错误|表示请求包含语法错误或无法被服务器处理，责任通常在客户端（如请求了不存在的资源）。|
|`5xx`|`500-599`|服务器错误|表示服务器在处理一个看似有效的请求时发生了内部错误，责任在服务器端。|

具体详见：[HTTP 响应状态码](https://developer.mozilla.org/zh-CN/docs/Web/HTTP/Reference/Status)
